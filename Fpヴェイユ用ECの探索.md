# $E(F_p)$ ペアリングフレンドリー曲線の見つけ方

<div style="text-align: right;">
2025/3/6 Daiji Sanai
</div>

---
Weil Pairingは、有限体$F_p$上の楕円曲線$E(F_p)$上でも成立する場合がある。
しかし、ペアリングが可能なな$E(F_p)$の曲線は、一般的ではないため、独自で発見しなければならない。
その曲線の条件と探索方法を記す。

### 1. ペアリング可能な$E(F_p)$の特徴

* ねじれ $r$ のサブグループが $r+1$ 個存在する。  
つまり、無限遠点$O$は各グループに共通の１点なので、  
$\#E=(r-1)(r+1)+1$ となる。(+1は無限遠点分)
* 曲線の位数は平方数になり、 $\#E=r*r$ となる。(コファクター $h=r$)

* 埋め込み次数 $k=1$である。($p \mod r = 1$)

### 2. 利便性の為のパラメータ制約

プログラミングや計算効率を向上させるための条件

* 素数 $p \mod 4 =3$ ... 平方剰余が計算で求められる条件
* 過去データから、素数 $p \mod 12 =7$ の時に$h=r$が成立している。
* 楕円曲線 $y^2=x^3+ax+b$ において、$a=0$
つまり、$EC: y^2=x^3+b$ 
* $k=1$ であれば、$b < 50$ 程度の小さな $b$ が存在することが多い（98％以上）。

### 3. アプローチ

* 必要とされる 素数 $r$ の大きさから、ターゲットとする 素数 $r$ を決める。
* 位数 $\#E=r*r$ から、楕円曲線の位数が決まる。
* 位数と基底素数$p$ の関係はハッセの定理から  
$ |\#E - (p + 1)| \leq 2\sqrt{p} $  
$ |r^2 - (p + 1)| \leq 2\sqrt{p} $
* この条件に合致（範囲内の）する 基底素数 $p$ を探索する。
* $p$ を見つけたら、$r=h$となるように、パラメータ $b$ をブルートフォーステストし、$b$ を決定する。
* $r$ に対応する $p$ が見つからなければ、$r$ の次の素数$(r_2)$を選び$r=r_2$とし、同じ探索を繰り返す。

### 4. 高速探索
$r$ を256bit級の素数としたい場合、基底素数 $p$ は512bit級になり、高速化の為に工夫が必要となる。

#### 4.1 $k=1$ とハッセの定理
ターゲットの素数rを決定すると、位数は、$\#E=r^2$ と決定される。これにより、基底素数 $p$ の探索範囲は、
ハッセの定理により、およそ次の範囲に限定される。
$$ r^2-2r < p < r^2 + 2r\\
r(r-2)<p<r(r+2)
$$

次に、$k=1$ である制約 ($p \mod r =1$) から、$p$ は $r$ の倍数＋１でなければならない。
$$p=mr+1$$
このことから、ハッセの定理範囲内にある$p$の候補は、
$$r(r-2)+1\\
r(r-1)+1\\
r^2+1\\
r(r+1)+1\\
r(r+2)+1
$$
の５つだけとなる。

ここで、$r$ は素数であることから、$p$ が奇数になる要素は、
$$r(r-1)+1\\
r(r+1)+1\\
$$
の２つだけということになる。

##### 基底素数 $p$ の候補は２点だけ
ねじれ素数 $r$を選定したときの基底素数 $p$ は、
$p_1=r(r-1)+1\\ 
p_2=r(r+1)+1$ 
の２つを素数判定すればよい。

#### 4.2 実用性$\mod 12=7$
$p$ の候補である $p_1,p_2$ の素数判定は、512bit級ともなると計算負荷が大きい。そこで、素数判定の前に、実用性のためのフィルタリングを行う。
素数 $6x+1$ が$\mod 4 = 3$ となるように、
$$p\mod12＝7$$
のフィルタを掛ける。
※$\mod 4 =3$ を選ぶ理由は、楕円曲線の演算中に必要となる平方剰余の演算を簡単に求めるための条件である。
$$a^{\frac{1}{2}}=a^{\frac{p+1}{4}}$$

#### 4.3 素数判定
$p_1,p_2$ のどちらかが素数であれば、その価を基底素数 $p$ に設定する。

#### 4.4 楕円曲線パラメータ $b$ の高速探索

楕円曲線のパラメータとして、$p,r,\#E,a(a=0)$は既に決定しているので、最後のパラメータ $b$ を探索する。探索する範囲は
$$3\le b\le 50 \space$$
程度である。経験的にこの程度の範囲に、条件を満たす $b$ は存在しているので、ブルートフォーステストで $b$ を探索する。

```pascal
function search_b(p,r: BigInt):BigInt;
var
    b: BigInt ;
    S,T : ECPoint ;
begin
    //ECパラメータ設定
    EC.p := p;
    EC.r := r;
    EC.a := 0;                  //a=0 で固定

    for b := 3 to 50 do         //必要時応じてレンジを拡大する
    begin
        EC.b := b;              //ECパラメータ b を書き換える
        S := getECPoint(EC);    //曲線(EC)上の点を１点見つける
        T := ECScalar(S,r);     //点Sをねじれr倍する
        if T = Infinity then    //無限遠点だったなら b 発見。
        begin
            result := b;
            exit;
        end;
    end;

    result := 0;                //見つからなかった
end.

```
