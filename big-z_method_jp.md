# Big-Z Method
# $E(\mathbb{F}_p)$ ペアリングフレンドリー曲線の見つけ方
---

Weil Pairingは、有限体 $\mathbb{F}_p$ 上の楕円曲線 $E(\mathbb{F}_p)$ 上でも成立する場合があります。
しかし、ペアリングが可能な $E(\mathbb{F}_p)$ の曲線は一般的ではないため、独自で発見しなければなりません。
その曲線の条件と探索方法を記します。

### 1. ペアリング可能な $E(\mathbb{F}_p)$ の特徴

* ねじれ $r$ のサブグループが $r+1$ 個存在する。  
    つまり、無限遠点 $O$ は各グループに共通の1点なので、曲線の位数 $\lvert E \rvert$ は次のようになります。

$$\lvert E \rvert = (r-1)(r+1) + 1$$

※ $+1$ は無限遠点分です。

* 曲線の位数は平方数になり、コファクターを $h=r$ とすると、次のようになります。

$$\lvert E \rvert = r \times r$$

* 埋め込み次数は $k=1$ であるため、次の条件を満たします。

$$p \pmod r = 1$$

### 2. 利便性の為のパラメータ制約

プログラミングや計算効率を向上させるための条件です。

* 素数について：

$$p \pmod 4 = 3$$

※これは平方剰余が計算で求められる条件です。

* 過去データから、次の条件の時に $h=r$ が成立しています。

$$p \pmod{12} = 7$$

* 楕円曲線 $y^2 = x^3 + ax + b$ において、 $a=0$ とします。つまり、次の曲線になります。

$$EC: y^2 = x^3 + b$$

* $k=1$ であれば、 $b < 50$ 程度の小さな $b$ が存在することが多いです（98％以上）。

### 3. アプローチ

* 必要とされる素数 $r$ の大きさから、ターゲットとする素数 $r$ を決めます。
* 位数 $\lvert E \rvert = r \times r$ から、楕円曲線の位数が決まります。
* 位数 $\lvert E \rvert$ と基底素数 $p$ の関係は、ハッセの定理から次のようになります。

$$| \lvert E \rvert - (p + 1) | \le 2\sqrt{p}$$

$$|r^2 - (p + 1)| \le 2\sqrt{p}$$

* この条件に合致（範囲内）する基底素数 $p$ を探索します。
* $p$ を見つけたら、 $r=h$ となるように、パラメータ $b$ をブルートフォーステストし、 $b$ を決定します。
* $r$ に対応する $p$ が見つからなければ、 $r$ の次の素数 $r_2$ を選び、 $r=r_2$ としておなじ探索を繰り返します。

### 4. 高速探索

$r$ を256bit級の素数としたい場合、基底素数 $p$ は512bit級になり、高速化の為に工夫が必要となります。

#### 4.1 $k=1$ とハッセの定理

ターゲットの素数 $r$ を決定すると、位数は $\lvert E \rvert = r^2$ と決定されます。これにより、基底素数 $p$ の探索範囲は、ハッセの定理によりおよそ次の範囲に限定されます。

$$r^2 - 2r < p < r^2 + 2r$$

$$r(r-2) < p < r(r+2)$$

次に、 $k=1$ である制約（ $p \pmod r = 1$ ）から、 $p$ は $r$ の倍数 $+1$ でなければなりません。

$$p = mr + 1$$

このことから、ハッセ的定理の範囲内にある $p$ の候補は、次の5つだけとなります。

$$r(r-2) + 1$$

$$r(r-1) + 1$$

$$r^2 + 1$$

$$r(r+1) + 1$$

$$r(r+2) + 1$$

ここで $r$ は素数であることから、 $p$ が奇数になる要素は、次の2つだけということになります。

$$r(r-1) + 1$$

$$r(r+1) + 1$$

##### 基底素数 $p$ の候補は2点だけ

ねじれ素数 $r$ を選定したときの基底素数 $p$ は、次の2つを素数判定すればよいです。

$$p_1 = r(r-1) + 1$$

$$p_2 = r(r+1) + 1$$

#### 4.2 実用性 $\pmod{12} = 7$

$p$ の候補である $p_1, p_2$ の素数判定は、512bit級ともなると計算負荷が大きいです。そこで、素数判定の前に実用性のためのフィルタリングを行います。

素数 $6x+1$ が $\pmod 4 = 3$ となるように、次のフィルタを掛けます。

$$p \pmod{12} = 7$$

※ $\pmod 4 = 3$ を選ぶ理由は、楕円曲線の演算中に必要となる平方剰余の演算を簡単に求めるための条件だからです。

$$a^{\frac{1}{2}} = a^{\frac{p+1}{4}} \pmod p$$

#### 4.3 素数判定

$p_1, p_2$ のどちらかが素数であれば、その値を基底素数 $p$ に設定します。

#### 4.4 楕円曲線パラメータ $b$ の高速探索

楕円曲線のパラメータとして、 $p, r$ 、および位数 $\lvert E \rvert$ 、 $a \ (a=0)$ は既に決定しているので、最後のパラメータ $b$ を探索します。

探索する範囲は次の通りです。

$$3 \le b \le 50$$

経験的にこの程度の範囲に条件を満たす $b$ は存在しているので、ブルートフォーステストで $b$ を探索します。

```pascal
function search_b(p,r: BigInt):BigInt;
var
    b: BigInt ;
    S,T : ECPoint ;
begin
    //ECパラメータ設定
    EC.p := p;
    EC.r := r;
    EC.a := 0;                  //a=0 で固定

    for b := 3 to 50 do         //必要時応じてレンジを拡大する
    begin
        EC.b := b;              //ECパラメータ b を書き換える
        S := getECPoint(EC);    //曲線(EC)上の点を１点見つける
        T := ECScalar(S,r);     //点Sをねじれr倍する
        if T = Infinity then    //無限遠点だったなら b 発見。
        begin
            result := b;
            exit;
        end;
    end;

    result := 0;                //見つからなかった
end.

```
